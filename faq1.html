为什么 npm 安装全局依赖包会出现错误？

执行 npm install PACKAGE -g 时在前面加上 sudo 即可解决。

为什么我的应用在命令行开启服务之后，使用「访问链接」生成的链接显示 Invalid Host header？

这是因为新版的 webpack-dev-server 出于安全考虑，默认检查 hostname ，如果 hostname 不是配置内的，将中断访问。您可以设置 webpack-dev-server 的 disableHostCheck 为 true。给命令添加 --disableHostCheck=true 或者给 webpack.config.js 添加如下配置：

devServer: {
 disableHostCheck: true
}

为什么在使用apt-get安装依赖包或升级环境会失败?

先更新下源sudo apt-get update 即可解决

为什么我创建工作空间（或插件）时会提示 system error？

如果您在创建工作空间或插件时遇到了下面的问题，可能是由于您修改过用户名导致的，这会导致您的信息不同步。请清除浏览器 cookie 并退出重新登录使用。
